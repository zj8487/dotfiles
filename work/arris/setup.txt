- install svn
sudo apt-get install subversion

- java stuff
sudo add-apt-repository ppa:webupd8team/java
sudo apt-get update && sudo apt-get install oracle-java7-installer

- install eclipse (in e.g. ~/programs/eclipse)

- use eclipse.ini from dotfiles

- install JavaHL library for eclipse (settings should be ok in eclipse.ini)
sudo apt-get install libsvn-java

- checkout code
cd ~/workspace/ && svn co https://dg-svns.arrisi.com/ws_phoenix/trunk phoenix
cd ~/workspace/ && svn co https://dg-svns.arrisi.com/ws_testportal/trunk testportal

- use h2 db with jetty (do not check in this file!)
echo 'svg.deploy.type = dev_h2' > ~/workspace/phoenix/dpcust_phoenix/build_dev.properties

- or if mysql
sudo apt-get install mysql-server mysql-client mysql-workbench
echo 'svg.deploy.type = dev > ~/workspace/phoenix/dpcust_phoenix/build_dev.properties

- run eclipse-plugins.sh

- install checkstyle from ~/workspace/phoenix/dreampark/checkstyle_medium.xml (TODO write description)

- install gradle plugin in eclipse and build model (TODO write description)

- create working sets to simplify navigation in eclipse

I don't think the following step is needed when running mysql
in dpcust_phoenix/db/data_scripts/mock_data.rb add the following line:
createCustomerWithBox("00:02:9b:4f:24:63", "test", "customer")

Migrating data run migrate_console.rb, for the testportal:
~/workspace/testportal/dreampark $ ./jruby/bin/jruby db/migrate_console.rb

In order to migrate you also need to tell dreampark about customer.home in
~/workspace/testportal/dreampark/build.properties
change customer.home and customer.name to

customer.home = ../dpcust_testportal
customer.name = dpcust_testportal

should obviously be dpcust_phoenix for the phoenix project

to mock epg data for e.g. testportal, put the following line:
ps.epgContentProvider = mock
in file ~/workspace/testportal/dpcust_testportal/web/WEB-INF/dev.properties
